LMT-Atlas

Documentation

Methods, definitions, and practical guidance for using the LMT-Atlas v1.0 static release without rewriting source evidence tables.

How To Use This Database

  • Use Search for enzyme names, species, UniProt, PDB, DOI, patent numbers, and exact provenance IDs.
  • Use Browse Enzymes to filter the full canonical enzyme/construct identity index.
  • Read the methanol-tolerance score, 95% interval, confidence, and fit status as the primary tolerance summary.
  • Read the public tolerance badge as a coarse stable-rank summary: More tolerant 37, Less tolerant 38, Uncertain 364.
  • Additional archive labels remain available only in downloads; they are not used for headlines or Browse/Search filtering.
  • Use the Assay Evidence Matrix on linked entries to inspect methanol concentration, incubation time, residual activity, and protocol grouping.
  • Because assay conditions are heterogeneous, filter by temperature, method, concentration, and time before comparing entries. Raw residual activity across conditions is not directly comparable.
  • Separate literature evidence, patent evidence, and process-confounded evidence when interpreting the source mix.
  • Download full tables or use the Export Wizard on Browse/Search to export the current filtered subset as continuous-score rows or conservative tiered labels.

Legacy representative enzyme page index: enzyme_pages/index.html.

Intrinsic Methanol Tolerance

Intrinsic methanol tolerance refers to free-enzyme methanol challenge evidence with explicit methanol concentration, exposure time or feeding mode, and a clearly defined activity or residual activity readout. Reactor-scale biodiesel outcomes are kept separate as process-confounded evidence.

Evidence Tiers

  • Strong direct evidence: free-enzyme methanol challenge data with usable assay context and provenance.
  • Process-confounded evidence: retained for context, not used as the main tolerance signal.
  • Mechanism or background evidence: useful for interpretation, not for tolerance scoring.

Official Score Display

Continuous LTS_ref primary. Public pages use the frozen LTS_ref score with a 95% interval and a confidence badge as the main tolerance summary. Lower LTS_ref means more methanol tolerant. Internal archive fields remain available in CSV and SQLite downloads for reproducibility.

The web headline uses frozen score, interval, confidence, and fit-status fields. The discrete Browse/Search view uses the public More tolerant / Uncertain / Less tolerant badge.

The public badge is a coarse confidence tier based on stable ranking, with More tolerant / Less tolerant / Uncertain counts of 37 / 38 / 364. It complements the continuous score and should not be read as an absolute interval classification.

  • Public badge: stable-rank confidence tier used for headline discrete counts, Browse/Search filters, and table scanning.
  • Archive labels: retained in downloads/SQLite for audit; many rows are unclear because confidence is low, the activity baseline is unreliable, sensitivity is unstable, or support is otherwise insufficient.

Score Inference

The official values come from a preregistered latent-score workflow with an independent Bayesian cross-check. The cross-check confirmed the frozen ranking for converged constructs.

The score is derived from residual-activity evidence only when baseline activity interpretation is reliable. Confidence is reduced when baseline quality, extraction method, evidence strength, missing context, or sensitivity checks are weak.

Assay Comparability

Because assay conditions are heterogeneous, users should filter by incubation temperature, assay method/substrate, methanol concentration, and exposure time before comparing entries. Cross-condition raw residual activity values are not directly comparable; the database exposes all measurement rows and conditions so users can construct comparable subsets.

The Export Wizard supports condition-filtered comparable subsets, optional tiered labels with conservative gating, and optional reverse-mutation augmentation for WT-variant ML training views. These exported labels are derived from frozen LTS_ref values and are not new official LTS labels.

Source Count Metrics

literature_source_count counts valid DOI entries in source_reference.csv with source_type=literature; this is the source-layer headline metric and current direct evidence source count. registry_literature_breadth counts distinct DOI-bearing Phase 1A registry sources across direct_intrinsic, process_confounded, mechanism_only, and background_only evidence; it is reported as registry breadth only and does not satisfy the 180 source-layer target. direct_intrinsic_source_count is the subset of distinct DOI-bearing publications that contribute direct_intrinsic literature rows to the main methanol-challenge evidence layer.

Curation SOP

The document-layer curation policy for this release is defined in the current SOP and applies the same inclusion, provenance, assay-context, and quality-control rules used for the downloadable bundle.

PDB anti-inheritance

Mutant and engineered construct rows do not inherit parent experimental PDB IDs by default. If no mutant-specific structure is confirmed, the page reports that no exact construct-specific PDB is available and records any parent-scaffold caveat separately.

Reviewer checklist

Release review checks cover representative page closure status, source-table immutability for derived pages, protocol-level disposition completeness, and integrity validation of the atlas source tables after each rebuild.

Release Boundaries

This release uses five manually reviewed representative enzyme pages: CALB, TLL, RML, BCL, and UM-1. The rest of the atlas is available at construct-level through the variant summary, evidence table, and downloads, but does not yet have hand-curated enzyme-species narrative pages.

Known limitation: latest newly entered constructs such as RML N-glyco, EstSP2, and JK-1 carry lower confidence and are mostly uncertain_or_intermediate; this is intentional evidence-quality disclosure, not a missing table render.
